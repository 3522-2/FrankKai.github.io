HEXO博客二次体验

发表于 2017-03-01 | 分类于 hexo

君子 “ 报仇 ” ，十年不晚！

　　第一次搭建HEXO博客的起因是：刚刚接触前端，那会儿应该连html标签名都不记得几个，但是在自己愣是照着同学推荐的教程，一点一点地搭了起来。第一次申请github，第一次下载git，第一次下载nodejs，第一次用npm…太多太多的第一次，我愣是用了周六周日两天把破博客搭建起来，期间不知道踩了多少坑，但最终还是成功的把博客托管在了github上。但是仅仅停留在搭建成功博客上，后续一直都没有写文章之类的，因为我觉得实在是太难了，hexo那么那么多命令，git也有那么多命令，一会儿出这错了一会儿出那错了，于是就选择了一个简书来记录自己的博客，其间也在同事的带领下买了阿里云学生虚拟机，也买了域名，准备彻彻底底从后端到前端搭建一个个人博客，但是最终卡在了自己的懒惰上面，因为域名需要备案，而备案则需要拍照，成本太高，再加上自己对后台一窍不通，就渐渐失去了这个想法。
　　在前端各种技术里摸爬滚打了一段时间后，渐渐也掌握了这些工具的使用，学习能力也快速提升。后端仅仅学了一点node，距离自己搭建博客的路还很长，于是重新准备启用强大的静态博客框架HEXO，再次把代码托管到github上。
目前静态部分基本搭建完成。
　　下一步准备在Github托管。
　　之后我会陆续之前在简书写的文章迁移到hexo博客上来。
　　然后我会继续跟着掘金大神学习webpack+es6+nodejs+vue+mongdb搭建博客，并且将学习过程中的收获和发现以博客的形式发布到博客上。

欢迎扫码关注我的微信公众号！
